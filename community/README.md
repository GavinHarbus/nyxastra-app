# NyxAstra Community Templates

> Community-maintained prompt templates for [NyxAstra](https://gavinschneestudio.com/products/nyxastra.html).
> Browse the gallery at **<https://gavinschneestudio.com/nyxastra/templates/>**.

This directory holds the **source of truth** for every template that ships
in the public gallery — both the official starter pack and contributions
from the community.

---

## How it works

```
community/
├── templates/                  ← Source: one folder per template
│   └── <slug>/
│       ├── template.json       ← Prompt body, variables, parameter preset
│       ├── cover.<png|jpg|webp>
│       └── meta.yml            ← Author, license, category, tags, models
│
├── submissions/                ← Inbox: drop .nyxtemplate files here in a PR
│
├── scripts/                    ← Lint, unpack, build pipeline (Python)
│
└── dist/                       ← Generated by CI (gitignored)
    ├── index.json              ← Master manifest (consumed by website)
    ├── templates/<slug>.nyxtemplate  ← Re-packed single-file downloads
    └── covers/<slug>.webp      ← Optimized previews for the gallery
```

The cover images are stored as **separate binary files** rather than
inline base64 — that keeps PR diffs reviewable, lets us swap cover
formats without rewriting JSON, and avoids bloating the git history.
The single-file `.nyxtemplate` users download is **rebuilt by CI** from
the same source on every change, so the schema is always current.

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full guide. Short version:

1. Design a template in NyxAstra and export it (`Right-click → Export`).
2. Open a PR that drops the `.nyxtemplate` file into `community/submissions/`.
3. CI lints, unpacks, and adds the result to your PR automatically.
4. A maintainer reviews, then merges.

If you don't use git, [open an issue][issue-template] with the
`.nyxtemplate` attached and a maintainer will help you submit it.

[issue-template]: https://github.com/GavinHarbus/nyxastra-app/issues/new?template=template_issue.yml

## Content policy

All submissions must follow [CONTENT_POLICY.md](CONTENT_POLICY.md).
Templates that promote hate, harassment, sexually explicit content, or
violate the OpenAI / Azure OpenAI usage policies will not be accepted.

## Local development

```sh
cd community
python3 -m venv .venv && source .venv/bin/activate
pip install -r scripts/requirements.txt

# Run all checks against every template in `community/templates/`
python3 scripts/build_community.py --check

# Build the dist/ artifacts (index.json, single-file downloads, optimized covers)
python3 scripts/build_community.py --build

# Process pending submissions (drop a .nyxtemplate into submissions/, then run)
python3 scripts/intake_submissions.py
```
