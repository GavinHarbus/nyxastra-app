# NyxAstra Community Templates

> Community-maintained prompt templates for [NyxAstra](https://gavinschneestudio.org/products/nyxastra.html).
> Browse the gallery at **<https://gavinschneestudio.org/nyxastra/templates/>**.

This directory holds the **source of truth** for every template that ships
in the public gallery — both the official starter pack and contributions
from the community.

---

## How it works

```
community/
├── templates/                  ← Source: one .nyxtemplate file per template
│   └── <slug>.nyxtemplate      ← Self-contained: prompt + variables + cover + community block
│
├── featured.yml                ← Maintainer-curated featured slug list
│
├── scripts/                    ← Lint and build pipeline (Python)
│
└── dist/                       ← Generated by CI (gitignored)
    ├── index.json              ← Master manifest (consumed by website)
    ├── templates/<slug>.nyxtemplate  ← Re-stamped single-file downloads
    └── covers/<slug>.webp      ← Optimized previews for the gallery
```

Each `.nyxtemplate` is a single self-contained JSON document exported
directly by NyxAstra. It carries the prompt body, variables, parameter
preset, an embedded base64 cover image, **and a small `community` block**
with just three things: author, license, and category.

The filename minus `.nyxtemplate` is the slug — contributors pick it
once, and CI never rewrites or moves the file. One PR adds one file and
gets one CI round; no bot rewrites are needed.

Featured templates are curated by maintainers in
[`featured.yml`](featured.yml); contributors can never self-promote.

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full guide. Short version:

1. Design a template in NyxAstra. Fill in the **Community Sharing**
   section in the editor (author, license, category).
2. Right-click the template → **Export…** to save the `.nyxtemplate` file.
3. Open a PR that drops the file into `community/templates/` with a
   clear, URL-safe filename.
4. CI lints in one round; a maintainer reviews and merges.

If you don't use git, [open an issue][issue-template] with the
`.nyxtemplate` attached and a maintainer will help you submit it.

[issue-template]: https://github.com/GavinHarbus/nyxastra-app/issues/new?template=template_issue.yml

All submissions must follow [CONTENT_POLICY.md](CONTENT_POLICY.md).

violate the OpenAI / Azure OpenAI usage policies will not be accepted.

## Local development

```sh
cd community
python3 -m venv .venv && source .venv/bin/activate
pip install -r scripts/requirements.txt

# Run all checks against every template in `community/templates/`
python3 scripts/build_community.py --check

# Build the dist/ artifacts (index.json, single-file downloads, optimized covers)
python3 scripts/build_community.py --build
```
